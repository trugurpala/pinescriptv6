# Contributing

Contributions are welcome! Here is how you can help.

## Adding to LESSONS_LEARNED.md

If you encounter and solve a Pine Script v6 error, please add it using this format:

```markdown
### [Error Title]
- **Error message:** Exact error text from TradingView
- **Cause:** Why does it happen?
- **Fix:** How to resolve it
- **Example:**
```pine
// Bad
...
// Good
...
```
---
```

## Improving Reference Files

Each file under `reference/` and `concepts/` covers a specific namespace.
If you find a missing function, incorrect signature, or better example:

1. Fork the repo
2. Edit the relevant file
3. Open a pull request with a clear description

## File Conventions

- Use Markdown (`.md`) for all documentation
- Code blocks must use ```pine syntax highlighting
- Keep examples minimal and runnable (`//@version=6` at the top)
- Do not add v5-only syntax
